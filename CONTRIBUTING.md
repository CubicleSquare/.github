# Contributing to Cubicle Square Technologies

Thank you for your interest in contributing!  
We follow clean engineering practices and expect all contributors to maintain the same standards.

---

## ✔️ How to Contribute

1. Fork the repository  
2. Create a feature branch:
  `feature/<ticket-id>-<short-description>`
3. Write clean, well-tested code  
4. Commit with Conventional Commit format  
5. Open a Pull Request with a clear description  

---

## ✔️ Commit Format (Conventional Commits)

- feat: add interview evaluation pipeline
- fix: resolve WebSocket reconnection bug
- chore: update dependencies
- docs: improve setup instructions
- refactor: clean up audio buffer logic


---

## ✔️ Pull Request Requirements

- All tests must pass  
- Linting must pass (Prettier, ESLint, Black, Flake8, etc.)  
- Include screenshots for UI changes  
- No production secrets or sensitive data  
- PR description must mention the issue/ticket  

---

## ✔️ Coding Standards

### Frontend
- TypeScript required  
- React Hooks + Functional Components  
- Use SWR / React Query for data fetching  
- Follow internal UI library design guidelines  

### Backend
- Prefer Django or Node.js (Fastify/Express)  
- Follow service boundaries & domain-driven design  
- Add/update tests for every feature  

---

## ✔️ Testing

- Minimum **70% test coverage** for new contributions  
- For critical services: integration + load tests  

---

## ✔️ Communication

- Use descriptive PR titles  
- Keep discussions respectful  
- Follow our Code of Conduct
